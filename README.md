# HMTL
## HTML 구조태그
* HML은 **웹문서**를 뜻한다.
* '태그' 웹문서에 들어가는 내용~제목 모든 요소는 `<태그>`로 묶여서 만들어진다.
## HTML 주의사항
* 파일명.HTML
* 모든 파일명, 폴더명은 영문 소문자 사용, 특수문자 금지, 공백금지, 한글 금지, 숫자 앞 금지
* start.html = 0
* start1.html = 0
* 1start.html = x (숫자앞)
* basic start.html = x (공백)
* basic_start.html = O (허용 특수문자 _ -)
* basic%start.html = x (허용 외 특수문자)
* 시작.html = x (한글)
## vsc 단축키
* 화면 분할 (좌우) : `컨트롤 + \`
* 화면 분할 (상하) : `컨트롤 +k` -> `컨트롤 + \`
* 찾기/바꾸기 : `컨트롤 + h`
* 같은거 한줄 복사 : `쉬프트 + 알트 + 방향키`
* 블럭 위치 옮기기 : `알트 + 방향키`
## HTML 단축키
* 'html:5' 탭 키 : 구조태그 생성
* 한줄 주석 : `컨트롤 +/`
* 블록 주석 : `쉬프트 + 알트 + a`

## 태그속성 class, id
### class
* `<태그 id="값"></태그>`
* 반복하면 안되는 단독 명칭일 경우
* 예) header, gnb, footer 등
## HTML 기초 문서 태그
* 블록태그 : w100%, 새로운 행, 크기가 큼
* 인라인태그 : 열(옆으로 나열), 크기는 내용만큼 인식
### 제목
* `h1~h6` : 블록태그, h1 대제목, h6 소제목, h레벨이 높을수록 검색어로 인식 될 확률 높음. SEO마케팅 목적
### 내용
* `p` : 블록, 한줄 ~ 여러줄 단락형태의 내용일 때
* `br` : 인라인, 줄바꿈할때 사용
* `em` : 인라인, 단순 문맥 강조,주로 p자식으로 사용
* `strong` : 인라인, 중요+심각성 강조, P자식 위주
* `sub` : 아래 첨자
* `sup` : 윗 첨자
* `del` : 인라인, 문자 줄 치기, 할인, 삭제 등을 
* `blockquete` : 블록, 긴 인용문
* `q` : 인라인, 짧은 인용문
* `code` : 인라인, 컴퓨터 명령어를 화면에 표시할 때
* `address` : 블록, footer 안 주소 및 연락처, 해당 서비스 주소만 포함
나타낼때 사용
### 레이아웃
* `div` : 블록, 그룹, 자식이 2개 이상이며 같은 방향일 때, 블록과 인라인을 모두 자식으로 가질 수 있음
* `span` : 인라인, 그룹(위 div와 같음), 보조(디자인), 인라인만 자식으로 가질 수 있음
* `hr` : 블록, 경계구분용 (디자인 테두리 용도 아님)
### 특수문자 태그
* `&copy;` : footer 저작권마크 ©
